# log-newest-first

A change log should read top-down newest-first; out-of-order dates suggest an append in the wrong place.

- **Category:** index-log
- **Default severity:** warn
- **Fixable:** no

Configured in [.okftool.yaml](/reference/configuration.md); see [validation](/reference/validation.md) for the spec layer.
